## Рабочий план проекта

### 1. Постановка задачи
- Изучить условие задачи и формат данных.
- Уточнить, что именно требуется предсказывать:
  - `at_least_one`
  - `at_least_two`
  - `at_least_three`
- Разобрать метрику качества и понять, как на результат влияют `cpm`, временное окно, площадки и аудитория пользователей.

### 2. Первичный анализ данных
- Посмотреть размеры таблиц и состав признаков.
- Проверить пропуски и особенности кодировки значений.
- Изучить распределения `cpm`, активность пользователей и популярность площадок.
- Проанализировать структуру `validate.tsv`: размер аудиторий, длину окон, число площадок.

### 3. Подготовка пользовательских признаков
- Использовать `users.tsv` и `history.tsv`.
- Для каждого пользователя собрать:
  - демографические признаки;
  - частоту показов;
  - число площадок;
  - временную активность;
  - сессионные признаки.
- Восстановить пользовательские сессии по правилу разрыва более 6 часов.

### 4. Подготовка признаков площадок
- Использовать `history.tsv`.
- Для каждой площадки посчитать:
  - число показов;
  - число уникальных пользователей;
  - средний и квантильные `cpm`;
  - интенсивность показов;
  - признаки повторяемости контактов.

### 5. Построение признаков кампании
- Для каждой строки из `validate.tsv` собрать итоговое описание кампании.
- Добавить прямые признаки:
  - `cpm`
  - длина окна
  - число площадок
  - размер аудитории
- Агрегировать пользовательские признаки по `user_ids`.
- Агрегировать признаки площадок по списку `publishers`.

### 6. Первый baseline
- Обучить простую модель на агрегированных признаках.
- Провести честную локальную оценку через кросс-валидацию.
- Зафиксировать baseline как стартовую точку для всех следующих улучшений.

Текущий ML-baseline:
- признаки: параметры кампании, агрегаты по аудитории из `users.tsv` и `history.tsv`, агрегаты по площадкам;
- модели: `Ridge`, `RandomForestRegressor`, `ExtraTreesRegressor`;
- текущий лучший результат на 5-fold CV: `ExtraTreesRegressor`, метрика `24.89`.

### 7. Улучшение модельного блока
- Перейти к более сильной табличной модели.
- Основной кандидат: `CatBoost`.
- Сравнить новую модель с baseline по локальной метрике.

### 8. Усиление признаков
- Добавить признаки совместимости аудитории с площадками.
- Добавить признаки относительной силы `cpm`.
- Улучшить признаки, связанные с сессиями и временным окном кампании.

### 9. Единый пайплайн
- Собрать общий сценарий для обучения, оценки и предсказания.
- Сделать отдельные режимы:
  - честная локальная проверка;
  - генерация итогового `result.tsv`.
- Обеспечить одинаковую логику построения признаков в train и inference.

### 10. Сравнение подходов
- Сравнить несколько версий решения:
  - простой baseline;
  - улучшенный ML.
- Выбрать лучшую модель по локальной метрике.

### 11. Финализация проекта
- Подготовить итоговый пайплайн.
- Зафиксировать лучшую метрику.
- Описать использованные признаки, модели и выводы по результатам работы.
